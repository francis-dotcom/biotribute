Design Brief

BioTribute — Visual Redesign Brief

Public Tribute + Owner Console

You are a senior product designer. Redesign the visual experience for BioTribute, a memorial tribute platform that helps families publish beautiful, shareable pages for loved ones who have passed. The product is fully functional today — your job is to propose a visual and UX redesign (layout, typography, color, spacing, component styling, information architecture) while preserving all existing functionality. Do not remove features or simplify the data model.

Product Overview

BioTribute helps families create living memorial pages where visitors can read a life story, view photos and videos, watch a funeral livestream, leave moderated messages, send condolence cards, and contribute financial support. Each tribute has a unique URL slug (e.g. /SirFemiOgini) and a password-protected owner console (e.g. /console/SirFemiOgini) where the family organizer manages all content.

Tagline (from marketing): "A place to remember, together."

Real example tribute: Sir Femi Ogini (/SirFemiOgini) — a live production page with burial countdown (target date June 11, 2026, Nigeria time), YouTube livestream, video memories, photo gallery, service poster, moderated guestbook, condolence popup, and bank donation details (Eco Bank). Use this as your primary design reference scenario. Life dates: 1947–2026.

Brand & Visual Identity

Core brand feel

  • Dignified, warm, memorial-appropriate — never flashy, never social-media casual
  • Green memorial theme on marketing/landing: sage/evergreen accents on cream/ivory backgrounds
  • Serif headings for names, section titles, and emotional weight; sans-serif for labels, kickers, metadata, and UI controls
  • Tone: reverent, gentle, family-centered — like a printed funeral program brought to life digitally

Current color systems (two visual worlds exist today — unify or harmonize them)

Marketing landing page (/):

  • Background: warm cream #faf6f0
  • Accent green: #5e7c6e (hover #4b6557)
  • Gold highlight: #c2a15e
  • Text: #2a322d / muted #6e6b64
  • Typography: Georgia serif headings, Inter/system sans body

Public tribute pages (themeable per tribute):

  • Default "Ivory Memorial" palette: warm ivory #f9f9fb, gold accent #c8a96a, soft violet #60557c
  • 11 selectable theme presets: Ivory Memorial, Sage Garden, Quiet Sky, Cool Amethyst, Midnight Remembrance, Warm Candlelight, Rose Remembrance, Coastal Pearl, Evergreen Hymn, Harvest Wheat, Monochrome Charcoal
  • Themes swap CSS variables (--bg, --gold, --violet-deep, etc.) — content stays, colors change
  • Optional theme rotation: public page can cycle through 2+ themes on an interval (e.g. every 24 hours)
  • Page shell: rounded card on soft gradient backdrop; hero/background portrait images as CSS variables
  • Typography: Iowan Old Style / Palatino / Baskerville serif; Avenir Next / Helvetica sans for UI labels

Owner console currently inherits tribute theme variables on a dashboard shell — feels like an admin overlay on the memorial aesthetic rather than a distinct product surface.

Design direction to explore

Unify the landing green-and-cream identity with the tribute page ivory-and-gold system into one coherent BioTribute design language. Tribute themes should feel like curated memorial palettes, not arbitrary color pickers. The console should feel like a calm, professional workspace — related to the public page but clearly "backstage."

User Personas

1. Family Organizer (primary console user)

  • Setting up a tribute under time pressure (often days before a funeral)
  • Needs to upload photos, paste a livestream link, write biography, configure donation details, and moderate guest messages
  • May not be tech-savvy; needs clear section labels and save feedback
  • Checks visitor counts and video engagement before/during the service
  • Wants to preview exactly what the public will see

2. Memorial Visitor (public page user)

  • Arrives via shared link, QR code on printed program, or social media
  • May be on mobile at the funeral venue
  • Wants to quickly find: service details, livestream, how to donate, where to leave a message
  • Emotionally vulnerable — UI must be calm, readable, and never pushy
  • May watch livestream, open video memories in new tab, submit a moderated message

3. Platform Admin (secondary)

  • Approves messages, reviews private family submissions
  • Uses same console with additional auth layer

Page 1: Public Tribute Display (/{slug})

A single long-scroll memorial page inside a rounded shell. All sections below exist today and must remain functional in your redesign. Sections can be toggled off by the organizer (gallery, poster, videos, livestream).

Global behaviors

  • Theme system: page colors come from selected tribute theme; hero and background portrait images overlay the shell
  • Visit tracking: page view count shown in hero (when available)
  • Share: native share API or clipboard copy
  • Markdown support: names, taglines, honorifics, and body copy support inline markdown
  • Mobile-first: many visitors are on phones at services

Kicker: "In Loving Memory"

Content (top to bottom):

  • Donate badge on portrait (opens donation details modal)

Section 2 — Sticky Action Bar

Three icon pill buttons (currently icon-only):

  3. Support Family — scrolls to support section

Shows brief share status toast ("Link copied" / error).

Design note: Icon-only buttons may be unclear to older visitors. Consider labeled pills or a bottom mobile action bar.

Section 3 — Messages & Memories

Kicker: "Messages & Memories" | Heading: "Share a Memory"

Passive prompt: After 30 seconds on page, a bottom toast asks "Would you like to leave a message for the family?" with "Leave a Message" / "Maybe later" (snoozes 24h). Must feel gentle, not intrusive.

Section 7 — Live Stream (optional, toggleable)

Functional Requirements (Do Not Break)

Your redesign must preserve:

  1. All section toggle visibility (gallery, poster, videos, livestream, condolence popup, countdown)
  2. Theme system with 11 presets + optional rotation on public page
  3. Moderated message flow (submit → pending → approve → public feed)
  4. Three support flows: tribute card, bank donation details, private family message
  5. Video upload (direct file to cloud storage) AND URL embed (YouTube/Vimeo/MP4)
  6. Livestream embed + thumbnail fallback modes
  7. Countdown to burial date (Nigeria timezone)
  8. Visit analytics and video open analytics in console
  9. Image upload for hero, background, gallery, poster, condolence card, video thumbnails, livestream thumbnail
  10. Mobile share, scroll-to-section navigation, modal overlays
  11. Spam protection on public forms (Turnstile)
  12. Auto condolence popup with reshow logic + message prompt toast timing
  13. Service-date livestream prioritization (June 10–11)
  14. Console auth gate (login required)

Current Pain Points & Design Opportunities

Use these as redesign drivers — not bugs to ignore:

  1. Two visual systems: Marketing landing (green/cream/Inter) vs tribute pages (ivory/gold/Iowan) vs console (tribute-themed dashboard). Feels like three different products. Unify into one BioTribute design system.
  2. Console is one endless scroll. Dozens of form cards in a single page with anchor links. Consider tabbed navigation, sidebar IA, or stepped wizard — while keeping all fields accessible.
  3. Hero clutter. Countdown ring + donate badge + share icon + view count + theme swatches all orbit the portrait. Simplify visual hierarchy.
  4. Popup fatigue. Condolence auto-popup (20s), message toast (30s), and action bar compete for attention. Propose a respectful engagement model.
  5. Icon-only action bar. Share / Message / Support pills have no text labels — accessibility and clarity issue for older visitors.
  6. Theme swatches on public hero. Four colored dots expose internal theme tokens to visitors — likely debug UI that should be removed or replaced.
  7. Console ≠ consumer-grade. Form-heavy, developer-adjacent card layout. Family organizers under funeral stress need clearer progress, section completion states, and preview-what-you-edit.
  8. Footer hardcoded. "For Ogini's Family" / "Ogigrid" — should be dynamic per tribute.
  9. Publish vs Save Draft ambiguity. Launch State section mentions publishing as a future step. Design could introduce a clear Published/Draft status indicator.
  10. Moderation on separate page. Works functionally but breaks flow. Consider inline moderation panel or notification-driven review UX.
  11. Gallery complexity. Marquee, drag-scroll, lightbox, mobile native scroll — powerful but visually dense. Simplify the gallery presentation.
  12. Video memories open externally. Thumbnails link out to YouTube/new tab rather than inline player (except livestream). Decide if inline playback improves the experience.

Deliverables Requested

Please provide:

  1. Design system overview — typography scale, color tokens (including 2–3 recommended default tribute themes), spacing, border radius, shadow language that unifies landing + tribute + console
  2. Public tribute page redesign — full-page mockup or section-by-section specs for /SirFemiOgini with all sections visible, plus mobile layout for funeral-day livestream viewing
  3. Owner console redesign — information architecture proposal (navigation model), dashboard summary, and key editor screens (details, images, media, theme, moderation)
  4. Component library highlights — hero, action bar, message cards, gallery strip, video tiles, livestream player, support section, modals/popups, form cards
  5. Engagement UX recommendation — how to handle condolence popup + message prompt + action bar without overwhelming grieving visitors
  6. Accessibility & mobile notes — readable type sizes, touch targets, contrast on all 11 theme presets, especially Midnight and Evergreen dark-ish variants

Constraints

  • Preserve all functionality listed above — this is a visual/UX redesign, not a feature cut
  • Memorial-appropriate tone always — no gamification, no bright social UI patterns, no aggressive CTAs
  • Works for Nigerian/African funeral context — bank transfer donations (not Stripe checkout), Nigeria-time countdown, large extended-family audience, mobile-first at venue
  • Theme presets must remain swappable — your design system should define how each of the 11 palettes maps to your new tokens
  • Reference tribute: Sir Femi Ogini, 1947–2026, with livestream, countdown to June 11 2026 burial, videos, gallery, service poster, and Eco Bank donation details
